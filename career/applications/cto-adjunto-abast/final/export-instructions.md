# Instrucciones de exportación

## Archivos finales

- `cv-print-es.html`: versión diseñada para imprimir o exportar a PDF.
- `cv-google-docs-es.md`: versión limpia para copiar en Google Docs.
- `application-email-es.md`: email corto para enviar la candidatura.

## Abrir el CV HTML

1. Abre `career/applications/cto-adjunto-abast/final/cv-print-es.html` en un navegador.
2. Revisa que el contenido se vea limpio antes de imprimir.
3. No hace falta conexión a internet: el HTML es autocontenido.

## Guardar como PDF en macOS

1. Con el HTML abierto en el navegador, pulsa `Cmd + P`.
2. En destino, elige `Guardar como PDF`.
3. Usa tamaño de papel `A4`.
4. Activa `Imprimir fondos` solo si el navegador lo requiere para conservar líneas o separadores sutiles.
5. Mantén la escala en `100%`. Si el contenido se va a una tercera página por poco, prueba `95%`.
6. Guarda el archivo como `Manuel_Gonzalez_Riojo_CV_CTO_Adjunto.pdf`.

## Qué revisar antes de enviar

- Que el PDF tenga aproximadamente 2 páginas.
- Que el nombre, email y LinkedIn sean correctos.
- Que no haya cortes raros entre secciones importantes.
- Que Deutsche Bank aparezca como empleo actual.
- Que CGI aparezca solo como contexto anterior de consultoría.
- Que el documento no mencione formación no terminada ni habilidades no demostradas como fortalezas.

## PDF vs Google Docs

Usa el PDF cuando envíes la candidatura por email, portal de empleo o recruiter.

Usa `cv-google-docs-es.md` si quieres pegar el contenido en Google Docs para hacer una versión editable, cambiar formato visual o exportar desde Google Docs.

## Qué enviar

Para una candidatura normal:

- Adjuntar `Manuel_Gonzalez_Riojo_CV_CTO_Adjunto.pdf`.
- Usar el texto de `application-email-es.md` como cuerpo del email.

Si piden texto editable:

- Usar el contenido de `cv-google-docs-es.md`.
- Mantener el PDF como versión visual principal siempre que sea posible.
