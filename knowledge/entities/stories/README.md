# Stories

This folder contains story entities.

## What Belongs Here

- One concrete professional event or example.
- Situation, problem, action, result, evidence, and skills demonstrated.
- Links to the relevant company, role, project, technologies, skills, and achievements.

## Naming Convention

Use a short lowercase slug based on the event:

```text
short-event-name.md
```

If needed, include a project or year to disambiguate:

```text
short-event-name-project-or-YYYY.md
```

## Relationship With Other Entities

Stories demonstrate behavior and judgment. They should reference projects, roles, and skills rather than restating all surrounding context. A story should stay focused on one event.
