# Technologies

This folder contains technology entities.

## What Belongs Here

- Programming languages, frameworks, tools, platforms, systems, technical methods, and technical domains.
- Context about how the technology was used.
- Links to projects, roles, stories, skills, publications, and talks.

## Naming Convention

Use the common lowercase technology slug:

```text
technology-name.md
```

When a technology has multiple names, use the canonical project or industry name and record alternatives in `aliases`.

## Relationship With Other Entities

Technologies should describe demonstrated usage, not generic documentation. They should link to the projects, roles, and stories where the technology appears.
