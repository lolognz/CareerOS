# Projects

This folder contains project entities.

## What Belongs Here

- Professional, academic, open-source, internal, research, entrepreneurial, or creative work units.
- Project context, business problem, objectives, responsibilities, architecture, technologies, challenges, impact, lessons, and evidence.
- Links to companies, roles, stories, skills, technologies, publications, talks, and achievements.

## Naming Convention

Use a stable lowercase slug:

```text
project-name.md
```

If the name is generic, add a company or domain qualifier:

```text
project-name-domain.md
```

## Relationship With Other Entities

Projects are central evidence units. They should link outward to technologies, skills, stories, and achievements instead of copying their full content. One project can support many generated outputs because different outputs can select different relationships.
