# Technologies

This folder contains structured technology entities.

## What Belongs Here

- Tools, programming languages, frameworks, platforms, methodologies, systems, and technical domains.
- Context about how the technology was used.
- References to projects, roles, stories, skills, education, publications, talks, and raw interview fragments.

## What Does Not Belong Here

- Generic documentation copied from vendor websites.
- Claims of expertise without supporting work.
- Project-specific implementation detail that belongs in `projects/`.
- Generated keyword lists.

## References

Technology entities should link to projects, roles, stories, and skills that show concrete usage.

## Why This Exists

Technologies help generators select relevant technical context without duplicating tool lists across every output.
